# Документация функций приложения E-dit Mobile

В этом файле описаны основные функции и инструменты приложения E-dit, а также файлы, в которых реализована их логика, и ожидаемый результат.

---

## 1. Базовые команды (Basic Commands)
Инструменты для общей обработки и форматирования текста.

Отвечающие файлы:
- **Логика**: `src/lib/commands/basic.ts`
- **UI**: `src/components/Commands/BasicCommands.tsx`
- **Hook**: `src/hooks/useEditor.ts` (`applyCommand`)

| Название | Что делает | Результат выполнения |
| --- | --- | --- |
| **Лишние пробелы** | Заменяет множественные пробелы и табы на один пробел. | Убирает "дыры" в тексте. |
| **NBSP** | Заменяет неразрывные пробелы (`\u00A0`) на обычные пробелы. | Решает проблемы с форматированием при копировании с веб-сайтов. |
| **Обрезать края** | Удаляет пробелы в начале и в конце каждой строки. | Каждая строка начинается и заканчивается только текстом. |
| **ВЕРХНИЙ** | Переводит весь текст в верхний регистр. | Текст становится ЗАГЛАВНЫМ. |
| **нижний** | Переводит весь текст в нижний регистр. | Текст становится строчным. |
| **Каждое Слово** | Делает первую букву каждого слова заглавной. | Полезно для заголовков (Title Case). |
| **Предложения** | Делает первую букву каждого предложения заглавной. | Корректирует пунктуацию начала предложений. |
| **К знакам** | Убирает пробелы перед знаками препинания (.,;:!?). | `Слово ,` превращается в `Слово,`. |
| **От знаков** | Добавляет пробел после знаков препинания, если его нет. | `Слово,Слово` превращается в `Слово, Слово`. |
| **Без пунктуации** | Полностью удаляет все знаки препинания. | Остаются только буквы и пробелы (опасно для структуры). |
| **Без скобок** | Удаляет содержимое внутри любых скобок `()`, `[]`, а также сами скобки. | Очищает текст от пояснений. |
| **Без пустых строк** | Удаляет лишние переносы, оставляя максимум один пустой абзац между текстами. | Текст становится компактнее. |
| **В одну строку** | Заменяет все переносы строк на пробелы. | Весь текст объединяется в один длинный абзац. |
| **По предложениям** | Добавляет перенос строки после каждого конца предложения. | Каждое предложение начинается с новой строки. |
| **Без нумерации** | Удаляет цифры с точками или скобками в начале строк. | Списки становятся обычным текстом. |

## 2. Suno Команды (Suno Commands)
Специализированные команды для обработки текстов песен (генерация мета-тегов Suno AI).

Отвечающие файлы:
- **Логика**: `src/lib/commands/suno.ts`
- **UI**: `src/components/Commands/SunoCommands.tsx`

| Название | Что делает | Результат выполнения |
| --- | --- | --- |
| **Очистить теги** | Удаляет пояснения внутри тегов. | `[Verse 1 | upbeat]` -> `[Verse 1]` |
| **Отбивка тегов** | Добавляет пустые строки до и после каждого `[тега]`. | Визуально отделяет куплеты друг от друга. |
| **Заглавные** | Делает первую букву каждой строки (кроме тегов) заглавной. | Строки песен выглядят аккуратнее. |
| **Только текст** | Полностью удаляет все `[теги]` из текста. | Оставляет только сами стихи. |
| **Структура** | Удаляет стихи, оставляя только `[теги]`. | Выдает "скелет" песни (напр., `[Intro] \n [Verse 1]`). |
| **Умные пробелы** | Убирает лишние пустые строки внутри куплетов. | Куплеты сгруппированы, теги четко над ними. |
| **CAPS теги** | Переводит все `[теги]` в верхний регистр. | `[verse]` -> `[VERSE]` |

## 3. Анализатор символов (Symbol Analyzer)
Интерактивная панель, собирающая спецсимволы, использованные в тексте, в "пилюли" под шапкой.

Отвечающие файлы:
- **UI**: `src/components/SymbolAnalyzer/SymbolPanel.tsx`
- **Отображение**: `src/App.tsx` (`displayedText`, `toggledSymbols`)

- **Что делает**: Автоматически парсит набранный текст (`debouncedText`) на наличие спецсимволов, Markdown-синтаксиса (`---`, ````) и элементов списков. Создает кнопки для каждого найденного символа.
- **Действие по клику**: При нажатии на кнопку-символ, этот символ добавляется в фильтр (т.е. "скрывается"). Он визуально удаляется изо всего текста (`displayedText`), а редактор переходит в режим `readOnly`. Повторное нажатие возвращает символ обратно.

## 4. Синхронизация (Peer-to-Peer Sync)
Функция мгновенной передачи текста между устройствами без сохранения на сервере (через PeerJS).

Отвечающие файлы:
- **Логика**: `src/hooks/useSync.ts`
- **UI (Кнопка)**: `src/components/Sync/SyncButton.tsx`
- **UI (Модалка)**: `src/components/Sync/SyncModal.tsx`

- **Что делает**: Генерирует уникальный Peer ID для вашего устройства. Позволяет подключиться к ID другого устройства.
- **Результат**: Как только устанавливается статус `connected`, любые изменения в `activeEditor.text` отправляются на второе устройство, и наоборот (`handleReceiveText`).

## 5. Инструменты Редактора (Editor Toolbar)
Нижняя панель управления и буфером.

Отвечающие файлы:
- **UI**: `src/components/Editor/EditorToolbar.tsx`
- **Логика истории**: `src/hooks/useEditor.ts` (`undo`, `redo`)

| Название | Метод / Файл | Результат выполнения |
| --- | --- | --- |
| **Назад (Undo)** | `undo()` / `useEditor.ts` | Отменяет последнее изменение текста, возвращая предыдущее состояние из стека `past`. |
| **Вперёд (Redo)** | `redo()` / `useEditor.ts` | Возвращает отмененное изменение из стека `future`. |
| **Вставить** | `handlePaste()` / `EditorToolbar.tsx` | Читает текст из системного буфера обмена (`navigator.clipboard.readText`) и добавляет в конец текущего текста. |
| **Копия** | `handleCopy()` / `EditorToolbar.tsx` | Копирует весь текущий текст в буфер обмена устройства. |
| **Стереть** | `clear()` / `useEditor.ts` | Полностью очищает текстовое поле (`setText('')`). |
| **Заметки** | `<NotesSheet />` / `App.tsx` | Открывает кастомный Bottom Sheet для ввода временных записей поверх экрана. |
